# Minimal Haskell FFI Example

Calls a C function from Haskell using `capi` FFI — no Cabal, just `gcc` and `ghc`.

## Files

| File | Description |
|------|-------------|
| `adder.h` | C header declaring `int add(int, int)` |
| `adder.c` | C implementation returning `x + y` |
| `main.hs` | Haskell entry point importing and calling `add` |

## Build & Run

```sh
gcc -c adder.c -o adder.o
ghc main.hs adder.o -o main
./main
# 4
```

## How it works

The `capi` FFI import references the header and function name:

```haskell
foreign import capi "adder.h add"
  c_add :: Int -> Int -> Int
```

GHC uses the header to verify the C signature at compile time. The compiled
object file `adder.o` is passed directly to `ghc` for linking — no build system
required.

## Inspecting compiled artifacts

### Symbols

```sh
nm adder.o          # C side: T add  (T = defined in text/code section)
nm main.o | grep add
# U add                                   (U = undefined, external reference)
# T ghczuwrapperZC0ZCmainZCMainZCadd      (capi wrapper generated by GHC)
```

The `ghczuwrapper...` symbol is the small C shim `capi` generates to call
`add` through its declared header type rather than raw ABI.

### Disassemble the capi wrapper

```sh
objdump -d main.o | grep -A 25 "ghczuwrapper"
```

The wrapper (~11 instructions) receives GHC's 64-bit arguments, truncates them
to `int` width to match the C ABI, calls `add`, then sign-extends the result
back to 64-bit for Haskell.

### GHC intermediate output

```sh
ghc -ddump-simpl -fforce-recomp main.hs adder.o -o main   # Core (desugared Haskell IR)
ghc -ddump-stg   -fforce-recomp main.hs adder.o -o main   # STG (before code generation)
ghc -ddump-asm   -fforce-recomp main.hs adder.o -o main   # Native assembly
```

`-fforce-recomp` is needed because GHC skips recompilation (and thus the dump)
when the source is unchanged. Output goes to stdout; redirect to a file:

```sh
ghc -ddump-simpl -fforce-recomp main.hs adder.o -o main > main.dump-simpl 2>&1
ghc -ddump-stg   -fforce-recomp main.hs adder.o -o main > main.dump-stg 2>&1
ghc -ddump-asm   -fforce-recomp main.hs adder.o -o main > main.dump-asm 2>&1
```

### What the dumps reveal

#### [main.dump-simpl](main.dump-simpl) — Core (desugared Haskell IR)

The entire program is only **30 terms**. The FFI call signature after
desugaring:

```
__ffi_static_ccall_safe ghczuwrapperZC0ZCmainZCMainZCadd
  :: Int# -> Int# -> State# RealWorld -> (# State# RealWorld, Int# #)
```

Key observations:
- Arguments are **unboxed** `Int#` (`2#`, `2#`) — GHC strips the `Int` box before the call
- The `IO ()` return type is just `State# RealWorld -> (# State# RealWorld, a #)` — a state-passing trick, no actual monad object exists at runtime
- The result `Int#` is immediately re-boxed with `GHC.Types.I# ds1` to produce a Haskell `Int`

#### [main.dump-stg](main.dump-stg) — STG (before code generation)

Note: `-ddump-stg` is deprecated; prefer `-ddump-stg-final`.

```
sat_sRO = \u []          -- updatable thunk (lazy, computed once)
  let { sat_sRN = \u []  -- inner thunk: the FFI call + boxing
    case __ffi_static_ccall_safe ... [2# 2# realWorld#] of
      (#,#) _ result -> GHC.Types.I# result
  } in GHC.Show.show $fShowInt sat_sRN
```

`\u` means **updatable thunk** — evaluated lazily and the result is cached
(memoized) on first force. So `show sum` is not evaluated until `putStrLn`
demands it.

#### [main.dump-asm](main.dump-asm) — Native assembly

The safe FFI overhead is visible in `sat_sRN_info`:

```asm
call suspendThread    ; release GHC capability → other threads can run
call *%r14            ; call add(2, 2)
call resumeThread     ; reacquire capability, restore heap/stack pointers
```

After the call, the result is heap-allocated:
```asm
movq $GHC.Types.I#_con_info,-8(%r12)  ; write Int# constructor tag
movq %rbx,(%r12)                       ; write value 4
leaq -7(%r12),%rax                     ; tagged pointer (tag bit = 1)
```

Using `unsafe` instead of safe would remove the `suspendThread`/`resumeThread`
pair entirely.
